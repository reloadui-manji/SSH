# SSH/SFTP Remote Explorer — 数据库（存储）设计说明书

## 1. 引言

### 1.1 编写目的

本文档描述 SSH/SFTP Remote Explorer 扩展的数据存储结构、持久化方案和配置格式。本扩展不使用传统关系型数据库，而是采用 VS Code 配置系统和文件存储作为数据持久化方案。

### 1.2 存储概述

| 存储类型 | 存储位置 | 用途 |
|----------|----------|------|
| VS Code 全局配置 | `settings.json`（User Settings） | 连接配置、全局设置 |
| VS Code 工作区配置 | `.vscode/settings.json` / `.ssh-sync.json` | 项目级同步配置 |
| VS Code globalState | Extension context globalState | 密码短语等安全存储 |

## 2. 连接配置（ConnectionProfile）

### 2.1 存储位置

VS Code User Settings（`settings.json`）中的 `ssh.connections` 数组。

### 2.2 数据结构

```json
{
  "ssh.connections": [
    {
      "id": "manual-my-server",
      "name": "my-server",
      "host": "192.168.1.100",
      "port": 22,
      "username": "deploy",
      "protocol": "sftp",
      "auth": {
        "type": "privateKey",
        "privateKeyPath": "/Users/manji/.ssh/id_ed25519",
        "passphrase": ""
      },
      "backend": "auto",
      "remotePath": "/var/www/app",
      "connectTimeout": 10000,
      "keepaliveInterval": 30000
    }
  ]
}
```

### 2.3 字段说明

| 字段 | 类型 | 必填 | 默认值 | 描述 |
|------|------|------|--------|------|
| `id` | string | 否 | `manual-{host}-{port}` | 唯一标识符 |
| `name` | string | 否 | `{username}@{host}` | 显示名称 |
| `host` | string | **是** | — | 服务器 IP 或域名 |
| `port` | number | 否 | `22` | SSH 端口号 |
| `username` | string | **是** | — | SSH 登录用户名 |
| `protocol` | enum | 否 | `sftp` | 传输协议：`sftp` / `scp` |
| `auth.type` | enum | 否 | `privateKey` | 认证类型：`password` / `privateKey` / `agent` / `keyboard-interactive` |
| `auth.password` | string | 条件 | — | 密码认证时使用 |
| `auth.privateKey` | string | 条件 | — | 内联私钥内容（PEM 格式字符串） |
| `auth.privateKeyPath` | string | 条件 | — | 私钥文件绝对路径 |
| `auth.passphrase` | string | 否 | — | 加密私钥的密码短语 |
| `auth.agent` | string | 条件 | — | SSH Agent socket 路径 |
| `backend` | enum | 否 | `auto` | 连接后端：`auto` / `ssh2` / `openssh` |
| `remotePath` | string | 否 | `""` | 远程根路径 |
| `localPath` | string | 否 | — | 本地映射路径 |
| `connectTimeout` | number | 否 | `10000` | 连接超时（毫秒） |
| `keepaliveInterval` | number | 否 | `30000` | 心跳间隔（毫秒） |

### 2.4 ID 生成规则

```
if (conn.id exists):
  id = conn.id
else:
  id = `manual-${host}-${port || 22}`
```

### 2.5 认证配置变体

#### 2.5.1 私钥认证

```json
{
  "auth": {
    "type": "privateKey",
    "privateKeyPath": "/Users/manji/.ssh/id_ed25519"
  }
}
```

#### 2.5.2 密码认证

```json
{
  "auth": {
    "type": "password",
    "password": "my-secret-password"
  }
}
```

#### 2.5.3 SSH Agent 认证

```json
{
  "auth": {
    "type": "agent",
    "agent": "/tmp/ssh-xxxxx/agent.12345"
  }
}
```

#### 2.5.4 键盘交互认证

```json
{
  "auth": {
    "type": "keyboard-interactive"
  }
}
```

## 4. 全局配置项

### 4.1 存储位置

VS Code User Settings（`settings.json`）。

### 4.2 配置结构

```json
{
  "ssh.connections": [],
  "ssh.autoUpload": false,
  "ssh.autoUploadDelay": 500,
  "ssh.watchExtensions": ["*"],
  "ssh.ignorePatterns": [".git", "node_modules", ".vscode", ".ssh-sync.json"],
  "ssh.defaultProtocol": "sftp",
  "ssh.showStatusBar": true,
  "ssh.remotePath": "",
  "ssh.concurrency": 4,
  "ssh.keyPassphrases": {
    "/Users/manji/.ssh/id_ed25519": "saved-passphrase-here"
  }
}
```

### 4.3 字段详细说明

| 配置键 | 类型 | 默认值 | 描述 |
|--------|------|--------|------|
| `ssh.connections` | `Array<ConnectionProfile>` | `[]` | 手动连接配置列表 |
| `ssh.autoUpload` | `boolean` | `false` | 保存时自动上传开关 |
| `ssh.autoUploadDelay` | `number` | `500` | 自动上传防抖延迟（毫秒） |
| `ssh.watchExtensions` | `string[]` | `["*"]` | 监听的文件扩展名列表 |
| `ssh.ignorePatterns` | `string[]` | `[".git","node_modules",".vscode",".ssh-sync.json"]` | 忽略的 glob 模式 |
| `ssh.defaultProtocol` | `"sftp" \| "scp"` | `"sftp"` | 默认传输协议 |
| `ssh.showStatusBar` | `boolean` | `true` | 是否显示状态栏 |
| `ssh.remotePath` | `string` | `""` | 默认远程根路径（所有连接共用） |
| `ssh.concurrency` | `number` | `4` | 并发文件传输数量 |
| `ssh.keyPassphrases` | `Record<string, string>` | `{}` | 私钥路径 → 密码短语映射 |

## 5. 项目级同步配置

### 5.1 存储位置

工作区根目录下的 `.ssh-sync.json` 文件。

### 5.2 数据结构

```json
{
  "connectionId": "manual-my-server",
  "remoteRoot": "/var/www/app",
  "sync": {
    "autoUploadOnSave": true,
    "uploadOnSaveDebounceMs": 500,
    "include": ["**/*.ts", "**/*.js", "**/*.html", "**/*.css"],
    "exclude": [".git", "node_modules", "dist", "*.log"]
  }
}
```

### 5.3 字段说明

| 字段 | 类型 | 必填 | 默认值 | 描述 |
|------|------|------|--------|------|
| `connectionId` | `string` | **是** | — | 目标连接 ID（对应 `ssh.connections` 中的 `id`） |
| `remoteRoot` | `string` | 否 | `"/"` | 远程同步根路径 |
| `sync.autoUploadOnSave` | `boolean` | 否 | `true` | 是否启用保存时自动上传 |
| `sync.uploadOnSaveDebounceMs` | `number` | 否 | `500` | 防抖延迟（毫秒） |
| `sync.include` | `string[]` | 否 | `["**/*"]` | 包含的文件模式 |
| `sync.exclude` | `string[]` | 否 | `[]` | 排除的文件模式 |

### 5.4 模式匹配规则

```typescript
// 支持的通配符模式
'*'        → 匹配所有文件
'**/*'     → 匹配所有文件（递归）
'**/*.ts'  → 匹配所有 .ts 文件（任意目录）
'*.log'    → 匹配当前目录下的 .log 文件
'node_modules' → 匹配包含 node_modules 的路径

// 匹配逻辑
function matchesPatterns(filePath, patterns):
  for pattern in patterns:
    if pattern is '*' or '**/*': return true
    if pattern startsWith '**/':
      if filePath endsWith pattern[3:]: return true
    if pattern startsWith '*.':
      ext = pattern[1:]
      if filePath endsWith ext: return true
    if filePath includes pattern: return true
  return false
```

## 6. 安全存储

### 6.1 密码短语存储

| 项目 | 说明 |
|------|------|
| 存储位置 | VS Code `globalState`（Memento） |
| 数据结构 | `Record<string, string>` — 私钥路径 → 密码短语 |
| 配置键 | `ssh.keyPassphrases` |
| 加密 | 由 VS Code 内部管理（平台密钥链） |

### 6.2 存储示例

```json
// ssh.keyPassphrases (存储在 globalState 中)
{
  "/Users/manji/.ssh/id_ed25519": "user-entered-passphrase",
  "/Users/manji/.ssh/id_rsa": "another-passphrase"
}
```

### 6.3 敏感数据生命周期

| 数据类型 | 存储位置 | 生命周期 | 清理时机 |
|----------|----------|----------|----------|
| 私钥内容 | 内存 | 连接建立期间 | `connect()` 成功后清空 `connectConfig.privateKey` |
| 密码 | 内存 | 连接建立期间 | `connect()` 成功后清空 `connectConfig.password` |
| 密码短语 | globalState | 持久化（用户手动清除） | 用户修改 settings 或手动清除 |
| 连接配置 | settings.json | 持久化（用户手动管理） | 用户删除对应条目 |

## 7. 配置迁移

### 7.1 版本兼容性

| 版本 | 变更 |
|------|------|
| 0.1.0 | 初始版本，支持基本连接配置 |
| 0.1.1 | 新增 `backend` 字段、SSH 证书支持、PTY 终端 |

### 7.2 向后兼容

- 旧版本未设置的字段使用默认值
- `id` 字段为空时自动生成
- `auth` 字段为空时默认使用私钥认证

## 9. 数据访问模式

### 9.1 读取

| 场景 | 数据源 | 频率 |
|------|--------|------|
| 扩展启动 | `settings.json` | 一次 |
| 配置变更 | `settings.json` | 事件触发 |
| 同步操作 | `.ssh-sync.json` | 按需 |
| 密码短语 | `globalState` | 连接时 |

### 9.2 写入

| 场景 | 数据目标 | 触发条件 |
|------|----------|----------|
| 添加连接 | `settings.json` | 用户操作 |
| 编辑连接 | `settings.json` | 用户操作 |
| 删除连接 | `settings.json` | 用户操作 |
| 保存密码短语 | `globalState` | 连接时用户输入 |
| 切换自动上传 | `settings.json` | 用户操作 |

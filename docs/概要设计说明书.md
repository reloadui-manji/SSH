# SSH/SFTP Remote Explorer — 概要设计说明书

## 1. 引言

### 1.1 编写目的

本文档描述 SSH/SFTP Remote Explorer 的整体架构设计、模块划分、数据流和关键设计决策，为详细设计和实现提供指导。

### 1.2 系统目标

构建一个 VS Code 扩展，提供以下核心能力：
- SSH 连接管理（多连接、多认证、持久化）
- 远程文件系统浏览（树形视图）
- 远程文件编辑（透明 FileSystemProvider）
- 本地-远程文件同步（自动/手动）
- 远程终端集成

### 2.2 架构模式

本系统采用 **分层架构 + 事件驱动** 模式：

| 层次 | 职责 | 核心组件 |
|------|------|----------|
| **表现层** | UI 交互、视图展示、命令注册 | 树视图提供者、状态栏、命令处理器 |
| **业务层** | 连接管理、文件操作、同步逻辑 | ConnectionManager、SyncEngine |
| **通信层** | SSH/SFTP 协议交互 | SshConnection、OpenSshConnection |
| **工具层** | 日志、路径处理、URI 解析 | Logger、path utils、uri utils |

事件驱动通过 `EventEmitter` 实现模块间解耦：
- `onProfilesChanged` — 连接配置变更
- `onConnectionStatusChanged` — 连接状态变更
- `onDidChangeTreeData` — 树视图刷新
- `onDidChangeFile` — 文件系统变更通知

### 3.2 模块职责详述

#### 3.2.1 extension.ts（入口）

- 初始化 Logger、ConnectionManager、StatusBarManager
- 注册 TreeDataProvider 和 FileSystemProvider
- 初始化 SyncEngine
- 注册所有命令
- 监听配置变更和连接状态变更
- `activate()` / `deactivate()` 生命周期管理

#### 3.2.2 core/protocol.ts

定义核心数据结构：
- `Protocol` — 传输协议枚举（SFTP / SCP）
- `ConnectionBackend` — 后端类型（auto / ssh2 / openssh）
- `AuthConfig` — 认证配置接口
- `ConnectionProfile` — 完整连接配置接口
- `ConnectionStatus` — 连接状态枚举

#### 3.2.3 core/connection.ts

`SshConnection` 类 — 基于 `ssh2` 库的 SSH/SFTP 连接实现：
- 连接建立与断开
- 加密密钥自动检测与密码短语管理
- SSH Agent 集成（证书认证支持）
- SFTP 文件操作（read/write/list/delete/rename/chmod）
- 目录递归上传
- 文件上传/下载（流式传输 + 进度回调）
- PTY 终端通道创建

#### 3.2.4 core/connectionManager.ts

`ConnectionManager` 类 — 连接生命周期管理：
- 从 VS Code settings 加载连接配置
- 监听 settings 变更自动重载
- 连接实例缓存（Map<profileId, RemoteConnection>）
- 后端自动选择（ssh2 vs openssh）
- 连接/断开操作封装
- 状态变更事件发布

#### 3.2.5 core/remoteConnection.ts

`RemoteConnection` 接口 — 连接操作抽象：
- 定义统一文件操作接口
- 屏蔽底层 ssh2 和 openssh 实现差异
- `wrapSshConnection()` 适配器函数

#### 3.2.6 providers/remoteExplorer.ts

`RemoteExplorerTreeProvider` — 树形数据提供者：
- 实现 `vscode.TreeDataProvider<RemoteTreeItem>`
- 根节点 = 连接列表
- 连接节点展开 = 远程根目录内容
- 目录节点展开 = 子目录/文件列表
- 监听连接状态/配置变更事件自动刷新

#### 3.2.7 providers/remoteFileSystem.ts

`RemoteFileSystemProvider` — 自定义文件系统提供者：
- 实现 `vscode.FileSystemProvider` 接口
- 注册 `ssh://` URI 方案
- 所有操作通过 `RemoteConnection` 接口委托给 SSH 连接
- 文件变更事件发布（支持 VS Code 编辑器实时更新）

#### 3.2.8 sync/syncEngine.ts

`SyncEngine` — 文件同步引擎：
- 自动上传：监听本地文件变更 → 防抖 → 上传到远程
- 手动同步：项目根目录 ↔ 远程目录
- 默认连接绑定（连接成功后自动设为同步目标）

#### 3.2.9 sync/fileWatcher.ts

`FileWatcher` — 基于 chokidar 的文件监听：
- 支持多路径监听
- 支持忽略模式配置
- 内置防抖机制（可配置延迟）

#### 3.2.10 commands/

命令处理器注册函数：
- `connectionCommands.ts` — ssh.connect, ssh.disconnect, ssh.addConnection, ssh.refresh
- `fileCommands.ts` — ssh.uploadFile, ssh.downloadFile, ssh.deleteFile, ssh.renameFile, ssh.createFolder, ssh.chmodFile, ssh.openFile
- `syncCommands.ts` — ssh.syncUp, ssh.syncDown, ssh.syncDiff, ssh.toggleAutoUpload

## 4. 数据流

### 4.1 连接建立流程

```
用户点击连接
    │
    ▼
ssh.connect 命令
    │
    ▼
ConnectionManager.connect(profileId)
    │
    ├── 检查后端选择（backendSelector）
    │   ├── ssh2 → new SshConnection()
    │   └── openssh → new OpenSshConnection()
    │
    ├── conn.connect()
    │   ├── 解析认证方式
    │   ├── 处理加密密钥/密码短语
    │   ├── 建立 SSH 连接
    │   └── 建立 SFTP 通道（惰性）
    │
    └── 连接成功 → 触发 onConnectionStatusChanged
                      │
                      ▼
              StatusBar 更新 + TreeView 刷新
```

### 4.2 文件编辑流程

```
用户在 VS Code 中打开远程文件
    │
    ▼
RemoteFileSystemProvider.readFile(resource)
    │
    ├── 解析 URI → connectionId + remotePath
    │
    ├── 获取已连接实例
    │
    └── conn.readFile(remotePath) → Buffer
                │
                ▼
        显示在编辑器中

用户修改并保存
    │
    ▼
RemoteFileSystemProvider.writeFile(resource, content)
    │
    └── conn.writeFile(remotePath, Buffer)
                │
                ▼
        触发 onDidChangeFile → 编辑器刷新
```

### 4.3 自动上传流程

```
本地文件保存
    │
    ▼
chokidar 检测到 change 事件
    │
    ▼
FileWatcher.debounce(filePath)
    │
    ▼（延迟后）
SyncEngine.uploadFile(localPath)
    │
    ├── 计算相对于 workspace 的路径
    │
    └── conn.uploadFile(localPath, remotePath)
                │
                ▼
        流式传输 → 完成
```

## 5. 关键技术决策

### 5.1 ssh2 作为主要 SSH 库

**决策**：使用 `ssh2` + `ssh2-sftp-client` 作为底层 SSH 通信库。

**理由**：
- 纯 Node.js 实现，无需本地依赖
- 成熟的 SFTP 支持
- 完整的认证方式支持（密钥、密码、agent、keyboard-interactive）
- 流式文件传输，支持进度回调

**替代方案**：openssh CLI 封装（用于证书认证场景，通过 `OpenSshConnection` 类）

### 5.2 VS Code FileSystemProvider 模式

**决策**：实现 `ssh://` 自定义文件系统提供者。

**理由**：
- VS Code 原生支持自定义 FileSystemProvider
- 编辑器自动使用远程文件系统，无需手动上传/下载
- 与 VS Code 的搜索、diff、多光标等功能无缝集成

### 5.3 连接抽象接口

**决策**：定义 `RemoteConnection` 接口，`SshConnection` 和 `OpenSshConnection` 分别实现。

**理由**：
- 屏蔽底层实现差异
- 支持后端自动选择
- 便于未来扩展其他协议

### 5.4 chokidar 文件监听

**决策**：使用 chokidar 而非 Node.js fs.watch。

**理由**：
- 跨平台一致性好
- 原生支持忽略模式
- 事件去重和聚合

## 7. 安全设计

### 7.1 敏感数据保护

| 数据类型 | 存储方式 |
|----------|----------|
| 私钥内容 | 仅内存中，连接后清空 |
| 密码 | 仅内存中，连接后清空 |
| 密码短语 | VS Code globalState 安全存储（可选保存） |
| 连接配置 | VS Code settings.json（不含敏感内容） |

### 7.2 日志安全

- 私钥内容在日志中清空（`connectConfig.privateKey = ''`）
- 密码/密码短语在日志中清空（`connectConfig.passphrase = ''`）
- 仅记录操作事件和错误摘要，不记录明文凭证

## 2. 核心类型定义

### 2.1 协议类型（protocol.ts）

```typescript
// 传输协议
enum Protocol {
  SFTP = 'sftp',   // SSH File Transfer Protocol
  SCP = 'scp',     // Secure Copy Protocol
}

// 连接后端
type ConnectionBackend = 'auto' | 'ssh2' | 'openssh';

// 连接状态
enum ConnectionStatus {
  Disconnected = 'disconnected',
  Connecting = 'connecting',
  Connected = 'connected',
  Disconnecting = 'disconnecting',
  Error = 'error',
}
```

### 2.2 认证配置（AuthConfig）

```typescript
interface AuthConfig {
  type: 'password' | 'privateKey' | 'agent' | 'keyboard-interactive';
  password?: string;          // 密码认证时的密码
  privateKey?: string;        // 内联私钥内容（PEM 格式）
  privateKeyPath?: string;    // 私钥文件路径
  certificatePath?: string;   // SSH 证书文件路径（CA 认证）
  passphrase?: string;        // 加密私钥的密码短语
  agent?: string;             // SSH Agent socket 路径
}
```

### 2.3 连接配置（ConnectionProfile）

```typescript
interface ConnectionProfile {
  id: string;                   // 唯一标识，格式：manual-{name}
  name: string;                 // 显示名称
  host: string;                 // 服务器地址
  port: number;                 // SSH 端口（默认 22）
  username: string;             // 登录用户名
  protocol: Protocol;           // 传输协议
  auth: AuthConfig;             // 认证配置
  backend?: ConnectionBackend;  // 连接后端（默认 auto）
  remotePath?: string;          // 远程根路径
  localPath?: string;           // 本地映射路径
  connectTimeout?: number;      // 连接超时（ms）
  keepaliveInterval?: number;   // 心跳间隔（ms）
  concurrency?: number;         // 并发传输数
  source: 'manual';             // 配置来源
}
```

#### 3.1.3 connect() 详细流程

```
connect(profileId):
  1. 从 profiles 获取 profile，不存在则抛异常
  2. 检查已有连接，已 Connected 则直接返回
  3. 调用 profileHasCertificate() 检测证书
  4. 调用 selectConnectionBackend() 选择后端
  5. 根据后端类型创建连接实例：
     - openssh → new OpenSshConnection(profile, logger)
     - 其他    → new SshConnection(profile, logger)
  6. 绑定 onStatusChange / onDisconnected 事件
  7. 调用 conn.connect()
  8. 存入 connections Map
  9. 返回连接实例
```

#### 3.1.4 loadFromSettings() 逻辑

```
loadFromSettings():
  1. 读取 ssh.connections 数组配置
  2. 读取 ssh.defaultProtocol、ssh.remotePath、ssh.defaultBackend
  3. 遍历 connections 数组：
     - 跳过无 host 或 username 的条目
     - 生成 id: conn.id || `manual-${host}-${port||22}`
     - 构造完整 ConnectionProfile（合并默认值）
     - 存入 profiles Map
  4. 若有有效连接，触发 onProfilesChanged
```

#### 3.2.3 connect() 认证处理流程

```
connect():
  1. setStatus(Connecting)
  2. 设置超时定时器（默认 10 秒）

  3. 【加密密钥处理】如果 auth.type === 'privateKey' 且无 passphrase：
     a. 读取私钥内容
     b. 检测是否加密（PEM: 检查 ENCRYPTED 标记；OpenSSH: 用 parseKey 尝试解析）
     c. 如果加密：
        - 先查找已保存密码短语（ssh.keyPassphrases 配置）
        - 已保存 → 直接使用
        - 未保存 → showInputBox 提示用户输入
          - 输入了 → 保存后使用
          - 未输入 → 回退到 SSH Agent
  4. 构造 ssh2 connectConfig

  5. applyAuth(connectConfig):
     case 'password':
       - config.password = auth.password
       - config.tryKeyboard = true

     case 'privateKey':
       - 读取私钥内容（inline 或文件路径或默认密钥）
       - 检测 SSH 证书（检查 <key>-cert.pub 文件）
       - 如果有证书 + SSH Agent → 使用 agent 认证
       - 否则 → config.privateKey = keyData
       - 检测密钥格式（OpenSSH/PEM-RSA/PEM-EC/PKCS8/PEM-DSA）
       - 验证密码短语（passphrase + parseKey 测试）
       - 加密密钥无密码短语 → 回退到 SSH Agent
       - 始终设置 tryKeyboard 作为回退

     case 'agent':
       - config.agent = auth.agent || SSH_AUTH_SOCK

     case 'keyboard-interactive':
       - config.tryKeyboard = true

  6. 注册 keyboard-interactive 事件处理（交互式认证）
  7. 注册 ready / error / close 事件监听
  8. 调用 client.connect(connectConfig)

  9. ready 事件：
     - 清除超时
     - 清空 connectConfig 中的敏感数据（privateKey, passphrase）
     - setStatus(Connected)
     - resolve()

  10. error 事件：
      - 清除超时
      - 清空敏感数据
      - setStatus(Error)
      - reject(error)
```

#### 3.2.4 createDirectory() 递归创建算法

```
createDirectory(remotePath):
  1. 标准化路径（去除多余 /，处理前导 /）
  2. 按 / 分割路径段
  3. 逐段处理：
     a. 构建当前级路径
     b. 尝试 mkdir
     c. 如果失败，stat 检查是否已存在
        - 已存在 → 视为成功（继续下一段）
        - 不存在 → 抛出错误
```

#### 3.2.5 deleteDirectory() 递归删除算法

```
deleteDirectory(remotePath):
  1. readdir 获取目录内容
  2. 如果为空 → rmdir 直接删除
  3. 否则：
     a. 对每个条目：
        - 目录 → 递归调用 deleteDirectory
        - 文件 → unlink 删除
     b. 全部删除后 → rmdir 删除本目录
```

## 6. 错误处理策略

| 场景 | 处理方式 |
|------|----------|
| 连接超时 | Promise.reject(Error)，showErrorMessage 提示用户 |
| 认证失败 | 记录详细错误信息（auth method、methods left），showErrorMessage |
| 文件不存在 | FileSystemProvider 抛出 FileNotFound |

# SSH/SFTP Remote Explorer — 需求说明书

## 1. 引言

### 1.1 编写目的

本文档旨在明确 SSH/SFTP Remote Explorer VS Code 扩展的功能需求和非功能需求，为开发、测试、验收提供依据。

### 1.2 项目背景

SSH/SFTP Remote Explorer 是一个 VS Code 扩展，允许开发者在 IDE 内直接连接远程服务器、浏览/编辑远程文件、执行终端操作，以及实现本地与远程之间的文件同步。它消除了频繁切换终端工具和 FTP 客户端的需求，将远程开发工作流整合到 VS Code 环境中。

### 1.3 适用范围

本扩展面向需要在 VS Code 中进行 SSH/SFTP 远程开发的开发者，支持以下远程操作：
- 连接管理（多服务器、多认证方式）
- 远程文件系统浏览与编辑
- 文件上传/下载/同步
- 远程终端集成

## 2. 总体描述

### 2.1 产品形态

VS Code 扩展（Extension），类型为 Activity Bar 视图容器 + 自定义文件系统提供者 + 终端集成。

### 2.2 用户角色

| 角色 | 描述 |
|------|------|
| 开发者 | 使用本扩展连接远程服务器，进行文件操作和终端交互 |

### 2.3 运行环境

| 项目 | 要求 |
|------|------|
| IDE | VS Code ≥ 1.96.0 |
| Node.js | ≥ 18（VS Code 内置） |
| 远程服务器 | 支持 SSH/SFTP 协议的 Linux/Unix 服务器 |
| 网络 | TCP 连接到远程 SSH 端口（默认 22） |

## 3. 功能需求

### 3.1 连接管理

#### 3.1.1 连接配置

| 需求 ID | FR-CM-001 |
|---------|-----------|
| 描述 | 支持通过 VS Code 设置（`ssh.connections`）手动配置连接 |
| 优先级 | 高 |
| 输入 | 连接名称、主机地址、端口、用户名、认证方式、远程根路径 |
| 输出 | 连接配置持久化到 VS Code settings.json |

#### 3.1.2 认证方式

| 需求 ID | FR-CM-003 |

| 描述 | 支持多种 SSH 认证方式：SSH 密钥、密码、SSH Agent、键盘交互 |

| 描述 | 支持加密私钥的密码短语管理（首次提示并保存） |
| 优先级 | 高 |

| 需求 ID | FR-CM-005 |

## 5. 配置项汇总

| 配置键 | 类型 | 默认值 | 描述 |
|--------|------|--------|------|
| `ssh.connections` | array | `[]` | 手动连接配置列表 |
| `ssh.autoUpload` | boolean | `false` | 保存时自动上传 |
| `ssh.autoUploadDelay` | number | `500` | 自动上传防抖延迟（ms） |
| `ssh.watchExtensions` | array | `["*"]` | 监听的文件扩展名 |
| `ssh.ignorePatterns` | array | `[".git","node_modules",".vscode",".ssh-sync.json"]` | 忽略的 glob 模式 |
| `ssh.defaultProtocol` | enum | `sftp` | 默认传输协议（sftp/scp） |
| `ssh.showStatusBar` | boolean | `true` | 显示状态栏 |
| `ssh.remotePath` | string | `""` | 默认远程根路径 |
| `ssh.concurrency` | number | `4` | 并发文件传输数 |
